# DD-0001: 毕业设计全流程落地路线图（方案二）

- Status: Implemented
- Date: 2026-02-25
- Author: 杨振国（草案由 Copilot 协助整理）
- Scope: `scheml_2` 项目

## 1. 背景与目标
当前项目已完成最小可运行闭环（初始化、基线运行、批量运行、时间戳产物、帕累托前沿可视化），但距离“可答辩的完整毕业设计”仍缺少：
1. 核心创新算子的工程化实现与验证；
2. 基线对比与消融实验体系；
3. 指标与统计显著性结论；
4. 文档、复现实验与论文图表一体化交付。

本 DD 目标是给出后续操作的可执行路线图，确保从“能跑”走到“可发表/可答辩”。

## 2. 问题定义与非目标
### 2.1 问题定义
围绕“基于协同拮抗与多目标优化的施肥养分推荐”完成端到端工程闭环：
- 数据层：可复用的数据适配、阶段划分、目标构造；
- 算法层：Beta 初始化 + 原型引导交叉 + 协同/拮抗耦合变异 + 动态精英保留；
- 评估层：HV/IGD/运行时间/资源开销 + 消融与对比；
- 产出层：可复现实验包、论文图表、答辩材料。

### 2.2 非目标（本阶段不做）
- 不在本阶段扩展多作物通用大平台；
- 不进行 UI 系统化开发；
- 不引入超大规模分布式训练框架。

## 3. 现状盘点（As-Is）
已完成：
- 工程骨架：`src/`, `experiments/`, `tests/`, `configs/`, `artifacts/`；
- 基线运行：`run_baseline.py`；
- 批量运行：`batch_run.py`；
- 产物规范：`final_population.csv`、`final_objectives.csv`、`pareto_front.png`、`manifest.csv`；
- 时间戳目录与引擎命名：`baseline_<engine>_<timestamp>`；
- DD 机制：`docs/design-docs/` + `copilot-instructions` 约束。

关键缺口：
- 真实数据与目标定义尚未完全固化；
- 核心创新算子尚未全部替换到主循环；
- HV/IGD 及统计分析尚未形成标准报告；
- 论文图表与代码产物尚未自动对齐。

## 4. 方案总览（To-Be）
采用“6 阶段里程碑”推进：

### 阶段 A：数据与任务定义冻结
目标：固定数据口径和实验任务边界，避免后续反复返工。
- A1. 确认数据源与字段：施肥时序（N/P/K）、产量、成本、环境指标；
- A2. 完成数据字典（字段名、单位、缺失处理、阶段划分规则）；
- A3. 固定训练/验证/测试划分规则与随机种子策略；
- A4. 输出 `data_spec.md` 与首版 `dataset_profile.csv`。

验收标准：
- 任意成员按文档可重建同样的数据切分与统计结果。

### 阶段 B：优化主循环升级（DEAP NSGA-II）
目标：将当前占位搜索替换为稳定的多目标进化主循环。
- B1. 接入 DEAP NSGA-II（保留现有 `Runner` 接口）；
- B2. 完成边界约束、编码解码、种群更新流程；
- B3. 为引擎提供统一日志与随机种子控制。

验收标准：
- 与当前 baseline 相同输入下可稳定复现；
- `pytest + smoke run` 全通过。

### 阶段 C：三大创新模块落地
目标：实现论文核心创新并可单独开关做消融。
- C1. Beta 连续偏向初始化（已基本可用，补参数敏感性）；
- C2. 原型引导交叉（KMeans 原型抽取 + 牵引退火）；
- C3. 协同/拮抗耦合变异（矩阵 M + 动态变异概率）；
- C4. 动态精英保留（综合距离、局部稀疏度、动态权重）。

验收标准：
- 每个模块有独立单元测试；
- 开关任一模块不破坏主流程；
- 消融脚本可一键运行。

### 阶段 D：代理模型与主动更新
目标：降低昂贵真值评估成本，形成“评估-更新”闭环。
- D1. LightGBM 代理训练与推理接口；
- D2. 每 G 代增量更新机制；
- D3. 不确定性采样策略（挑选真值评估样本）；
- D4. 记录代理误差与更新开销。

验收标准：
- 相比纯真值评估显著降低总耗时；
- 误差与性能曲线可视化齐全。

### 阶段 E：实验矩阵、对比与消融
目标：形成可答辩证据链。
- E1. 主对比：改进算法 vs NSGA-II baseline；
- E2. 消融：去初始化/去先验算子/去动态精英；
- E3. 指标：HV、IGD、运行时间、资源开销；
- E4. 多随机种子重复 + 统计显著性检验。

验收标准：
- 形成主结果表 + 消融表 + 关键图（前沿图、收敛曲线、箱线图）。

### 阶段 F：论文与答辩交付
目标：实现“代码-实验-论文图表”一致。
- F1. 固化图表导出脚本（论文可直接引用）；
- F2. 汇总方法章节与实验章节证据；
- F3. 准备答辩演示流程（问题->方法->结果->贡献->局限）。

验收标准：
- 从仓库可一键复现实验核心结果；
- 论文中关键数字可追溯到具体实验目录与 manifest。

## 5. 验证与评估计划
### 5.1 工程验证
- 单元测试：算子边界、概率范围、输入输出维度、异常路径；
- 集成测试：`run_baseline.py`、`batch_run.py`；
- 回归测试：固定 seed 下结果波动范围检查。

### 5.2 算法评估
- 指标：HV、IGD、运行时间、代理更新频次与存储占用；
- 可解释性：NPK 轨迹形态、原型牵引效果、协同拮抗方向性。

### 5.3 统计结论
- 至少 5 次随机种子重复；
- 报告均值±标准差，必要时做显著性检验。

## 6. 风险与回滚
主要风险：
1. 数据字段缺失或口径不一致；
2. 代理模型偏差导致优化方向漂移；
3. 算子耦合过强造成多样性下降；
4. 实验矩阵过大导致时间不可控。

回滚策略：
- 对每个阶段保留“基线可运行分支”；
- 关键模块特性开关化（可禁用）；
- 每阶段完成后冻结一版配置与产物目录。
